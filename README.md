# 珑临公馆 KTV订房管理系统

一个简洁高效的KTV房间预定管理系统，支持管理员和经理两种角色。

## 功能特性

- ✅ **用户权限管理** - 管理员预设经理账号，经理才能登录预定
- ✅ **房间状态展示** - 实时显示房间空闲/已预定状态
- ✅ **在线预定** - 经理可选择日期、时间段预定房间
- ✅ **预定管理** - 经理只能取消自己预定的房间
- ✅ **账号管理** - 管理员可添加、编辑、删除经理账号
- ✅ **响应式设计** - 支持手机和电脑访问
- ✅ **数据持久化** - 使用 SQLite 数据库存储

## 快速开始

### 环境要求

- Node.js 14.x 或更高版本

### 安装步骤

1. **克隆项目**（如使用 Git）

```bash
git clone <repository-url>
cd KTV订房系统
```

2. **安装依赖**

```bash
npm install
```

3. **启动服务器**

```bash
npm start
```

4. **访问系统**

打开浏览器访问: http://localhost:3000

## 使用说明

### 登录

1. 输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后自动跳转到房间列表

### 预定房间

1. 选择预定日期
2. 点击空闲房间的"立即预定"按钮
3. 填写预定信息（时间段必填，其他选填）
4. 点击"确认预定"

### 取消预定

1. 在"我的预定"标签页查看所有预定
2. 点击"取消预定"按钮
3. 确认取消操作

### 管理经理账号（管理员）

1. 点击"账号管理"标签页
2. 点击"添加经理"新增账号
3. 点击"编辑"修改账号信息
4. 点击"删除"移除账号

## 数据存储

系统使用 **SQLite** 数据库存储数据，具有以下优势：

- ✅ **持久化存储** - 数据存储在 `data/ktv.db` 文件中
- ✅ **无需配置** - 自动创建和初始化
- ✅ **跨平台兼容** - 支持所有操作系统

### 数据表结构

| 表名 | 说明 |
|------|------|
| rooms | 房间信息 |
| users | 用户账号（管理员和经理） |
| bookings | 预定记录 |

## 云端部署（Render）

详见 [DEPLOY_RENDER.md](./DEPLOY_RENDER.md)

### 快速部署步骤

1. **上传到 GitHub**
   - 创建新仓库
   - 上传项目文件（不含 node_modules）

2. **连接 Render**
   - 访问 [render.com](https://render.com)
   - 使用 GitHub 账号登录
   - 连接你的仓库

3. **创建 Web Service**
   - Build Command: `npm install`
   - Start Command: `npm start`
   - 选择 Free 实例类型

4. **完成部署**
   - 自动部署后获得访问 URL
   - 示例：`https://your-app.onrender.com`

### 部署后访问

- 首次访问可能需要 10-30 秒唤醒服务（免费版休眠机制）
- 数据自动持久化到 SQLite 数据库
- 更新代码只需推送到 GitHub

- **后端**: Node.js + Express

## 常见问题

### Q: 首次访问很慢？
A: 这是正常的！Render 免费版在无活动 15 分钟后会进入休眠，首次访问需要唤醒。

### Q: 数据会丢失吗？
A: 不会！SQLite 数据库文件保存在持久化存储中，重启后数据保留。

### Q: 如何更新代码？
A: 修改本地代码后推送到 GitHub，Render 会自动检测并重新部署。
